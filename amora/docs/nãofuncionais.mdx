# Requisitos não funcionais

Essas são os requisitos dos sistemas que se referem às propriedades dos sistemas e não às suas funcionalidades: 

## RNF01 – Usabilidade e interface 

- RNF01.01 - A interface deve seguir a identidade e o sistema de design corporativo (Bosch Brand Guide), garantindo padronização e familiaridade para os colaboradores; e 

- RNF01.02 - As atualizações de valores em euros e o somatório de horas no formulário de solicitação devem ocorrer de forma instantânea (tempo de resposta inferior a 2 segundos). 
 

## RNF02 – Segurança e controle de acesso 

- RNF02.01 - O sistema deve limitar as ações de acordo com o papel do usuário. Apenas colaboradores com perfil de administrador podem mover os cards no painel Kanban ou visualizar a listagem completa de solicitações de outros setores. 
 

## RNF03 – Desempenho 

- RNF03.01 - O painel Kanban deve ser capaz de carregar e renderizar os cards de solicitação de forma fluida, sem travamentos ou atrasos perceptíveis durante a movimentação por drag and drop ou atualização de status. 
 

## RNF04 - Integridade dos dados 

- RNF04.01 - O banco de dados deve garantir a persistência correta dos dados (como centro de custo, prazos definidos e valores calculados) para evitar perda de dados em atualizações ou reinicializações do sistema.
